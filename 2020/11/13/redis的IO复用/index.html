Redis 基于 Reactor 模式开发了自己的网络事件处理器，使用 I/O 多路复用程序来同时监听多个套接字，并将到达的
事件传送给文件事件分派器，分派器会根据套接字产生的事件类型调用相应的事件处理器。

事件的调度与执行

服务器需要不断监听文件事件的套接字才能得到待处理的文件事件，但是不能一直监听，否则时间事件无法在规定的
时间内执行，因此监听时间应该根据距离现在最近的时间事件来决定。
